## 🎯 Quick Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Ready for deployment"
git push origin main
```

### 2. Deploy Backend (Choose One)

#### Option A: Railway (Recommended)
1. Go to [railway.app](https://railway.app)
2. Sign in with GitHub
3. "New Project" → "Deploy from GitHub repo"
4. Select your `mpt-mvp` repository
5. ✅ Done! Copy the generated URL

#### Option B: Vercel
1. Go to [vercel.com](https://vercel.com)
2. Sign in with GitHub
3. "Import Project" → Select `mpt-mvp`
4. ✅ Done! Copy the generated URL

#### Option C: Render
1. Go to [render.com](https://render.com)
2. "New" → "Web Service"
3. Connect GitHub → Select `mpt-mvp`
4. Build Command: `npm install`
5. Start Command: `npm start`
6. ✅ Done! Copy the generated URL

### 3. Update Configuration
1. Edit `config.js` in your GitHub repository
2. Replace `'https://your-backend-url.railway.app'` with your actual URL
3. Commit and push the change

### 4. Enable GitHub Pages
1. Go to your repository → Settings → Pages
2. Source: "Deploy from a branch"
3. Branch: `main` / Folder: `/ (root)`
4. ✅ Save and wait 2-3 minutes

## 🔧 Troubleshooting

### Backend Issues
- **Status Page**: Visit `/status.html` for diagnostics
- **Console**: Press F12 → Console tab for error details
- **CORS**: Ensure your GitHub Pages URL is in server CORS config

### Data Not Saving
- **Green Status**: Should save to backend
- **Red Status**: Saves locally, syncs when backend returns
- **Check Console**: Look for specific error messages

### GitHub Pages Not Updating
- **Actions Tab**: Check for deployment failures
- **Cache**: Hard refresh with Ctrl+Shift+R
- **Wait**: Initial deployment can take 5-10 minutes
